## 二、模型物理属性与动力学参数 (Mass & Inertia)

在物理仿真中，刚体的质量与转动惯量 (Moment of Inertia) 决定了其受力后的加速度与物理稳定性。Gazebo 无法仅凭 STL 表面网格文件自动计算质量分布，因此所有参数必须在 SDF 中精确配置。

### 2.1 无人机主体 (base_link) 动力学参数

无人机空机质量（不含机械臂）设计为 3.6 kg。根据 550mm 轴距与对称质量分布估算的转动惯量矩阵如下：

```
          [Ixx  Ixy  Ixz]   [0.045  0       0    ]
I_drone = [Iyx  Iyy  Iyz] = [0      0.045   0    ] kg·m²
          [Izx  Izy  Izz]   [0      0       0.071]
```

微雪官方提供的原始机械臂 URDF 总重为 0.63553 kg。为了对齐真实测量的 0.85 kg 目标质量，必须在不改变几何质心的情况下，将机械臂所有连杆的质量和转动惯量等比乘以放大系数：

## 三、空间坐标对齐与旋转轴修正

### 3.1 尺寸冲突（毫米与米换算）

- **病因**：ug 软件导出的 STL 模型默认单位为毫米 (mm)，而 Gazebo 的标准物理单位为米 (m)。直接导入会导致无人机体积变大 1000 倍。
- **解决方案**：在 SDF 的 base_link 标签中，将 `<visual>` 内的 `<mesh>` 标签添加 `<scale>0.001 0.001 0.001</scale>`。

### 3.2 级联姿态调整 (relative_to 级联技术)

为避免每次微调安装高度都需要手动重新计算机械臂所有连杆绝对坐标的繁琐，采用级联坐标引用：

- 将机械臂底座 arm_base_link 的 `<pose>` 绑定在 base_link 上：

```xml
<pose relative_to="base_link">0.006 0.037 0.07 3.14159265 0 0</pose>
```

- 机械臂所有后续连杆的 `<pose>` 均采用相对于前级 Parent Link 的方式级联定义（例如 `relative_to="link1"`）。
- **效果**：调节安装位置或进行 180° 翻转挂载时，只需修改 arm_base_link 的一个 `<pose>` 标签，整个机械臂及关节会随之整体移动或翻转，无需更改任何下级坐标。

### 3.3 link2_to_link3 旋转轴物理修正

- **冲突表现**：修改 link3 的相对姿态后，机械臂在旋转该关节时发生严重扭曲或向错误方向运动。
- **数学原理解析**：
  在原始设计中，大臂 (Joint 2) 与小臂 (Joint 3) 的物理俯仰旋转轴是平行的。然而，为了满足特定的初始折叠需求，link3 相对于 link2 的相对姿态被修改为了：

```
Pose = [x, y, z, 0, -1.5707963, -1.5707963]
```

  经过 Roll-Pitch-Yaw 旋转矩阵 (RPY 坐标轴偏转) 换算后，原本平行于 link2 俯仰轴的局部 Z 轴在 link3 局部坐标系下，已经完全转到了 link3 的局部 X 轴方向。

- **修正方案**：必须将 link2_to_link3 关节中的旋转轴向量由默认的 Z 轴修改为 X 轴：

```xml
<axis>
  <xyz>1 0 0</xyz>  <!-- 经三维旋转矩阵换算，物理旋转轴已偏转至 local X 轴 -->
</axis>
```

## 四、物理稳定性调优（避免穿模、振颤与爆飞）

当模型在 `<static>false</static>` 状态下接触地面时，极易因约束冲突导致物理求解器计算溢出 (NaN) 进而弹飞。我们通过多重物理手段解决了这些问题。

### 4.1 起落架底部"四脚独立球体碰撞体"方案（简化计算方案）

- **传统 STL 碰撞的弊端（保留有roarm_quad_stl方案）**：直接使用复杂的机身 STL 网格 (drone_body.STL) 作为 `<collision>` 体，一方面会导致物理引擎构建 AABB 碰撞树时 CPU 过载假死（无响应）；另一方面，机身中心大面积碰撞箱极易与腹部的折叠机械臂发生穿模，同时接触地面时容易因微小的模型缺陷导致无人机站立歪斜或晃动。RTL（实时率过低）=0.07。
- **轻量化独立球体设计**：
  彻底剔除复杂的 STL 碰撞，在 4 个电机下方起落架末端的精确 xy 投影坐标处，定义 4 个直径为 6 cm 的极轻量化物理承重球 (Sphere Primitive)：
  - FR (右前): [0.152, -0.124, -0.19]
  - BL (左后): [-0.175, 0.202, -0.19]
  - FL (左前): [0.152, 0.202,  -0.19]
  - BR (右后): [-0.175, -0.124, -0.19]
- **效果**：球体碰撞检测极其轻量，Gazebo 加载速度提升数倍，且 4 点支撑提供了完美的刚性水平平衡，彻底消除了模型的倾斜与站立抖动。

### 4.2 基于 collide_bitmask 的内部自碰撞过滤

- **物理冲突**：当机械臂在无人机腹部折叠时，由于折叠紧凑，机械臂各连杆难免会伸入无人机机身的物理空间内（穿模）。Gazebo 会在这些不直接相连的非相邻连杆之间计算强烈的排斥力，导致模型内部激烈冲突并爆炸解体。
- **过滤方案**：在各 `<collision>` 标签中引入 `collide_bitmask`（碰撞掩码），通过二进制按位与（AND）运算控制碰撞关系：
  - 无人机起落架碰撞体掩码设为：0x01（二进制 01）
  - 机械臂所有连杆碰撞体掩码设为：0x02（二进制 10）
  - 地面及外界障碍物掩码保持默认：0xFF（二进制 11）
- **计算机制**：
  - 无人机与机械臂：0x01 & 0x02 = 0x00 → 不发生碰撞（完美腹部穿模，无自爆内力）。
  - 无人机与地面：0x01 & 0xFF = 0x01 → 正常触地站立。
  - 机械臂与地面/外界：0x02 & 0xFF = 0x02 → 正常发生物理抓取或撞击支撑。

### 4.3 消除 hand_tcp 虚拟节点的数值鞭动 (Numerical Whipping)

- **病因分析**：虚拟末端 hand_tcp 质量极轻（10 g），局部转动惯量极小（10⁻⁸），但通过一个长达 28 cm 的杠杆刚性连接在 link3 底部。由于杠杆平移转动惯量与局部转动惯量相差近 80,000 倍，导致物理引擎在迭代解算固定关节 (Fixed Joint) 约束时矩阵严重失调 (Ill-conditioned)，产生极高频的剧烈振颤（黑色质心球狂甩），最终带着整架无人机疯狂乱飞。
- **解决方案**：
  - **方案 A（当前使用）**：在 Gazebo 的 SDF 文件中直接删除 hand_tcp 这一 Link 及其 Fixed 关节。因为 hand_tcp 仅供 ROS 2 运动规划（MoveIt!/TF）使用，而 ROS 2 只读取 URDF，Gazebo 不需要这个无碰撞无视觉的虚拟节点。将其从 SDF 中删除后，物理发散源被彻底消灭，模型极其稳定。
  - **方案 B（已废弃）**：如果必须保留，应将其局部转动惯量人工放大至 1e-04，以匹配悬臂尺度，确保求解器数值收敛。

## 五、关节阻尼、摩擦力与虚拟拉簧机制 (Dynamics)

当关闭静态模型（`<static>false</static>`）且尚未给电机通电时，机械臂在重力作用下会自然坠落。由于没有缓冲，关节在猛烈撞击限位 (Limit) 的瞬间会产生无穷大内力，导致物理引擎发散（爆飞）。

### 5.1 关节阻尼与摩擦力定义

在各旋转关节的 `<axis><dynamics>` 中定义物理阻尼：

- **damping（粘性阻尼）**：提供与运动速度成正比的粘滞阻力 。在机械臂自由下落时起到类似"液压缓冲器"的效果，减缓撞击。
- **friction（干摩擦力）**：提供恒定的静摩擦阻力 ，提供机械臂关节一定的"自锁"能力。

### 5.2 隐式弹簧阻尼器保护

在各旋转关节中加入以下配置，开启 ODE 求解器的隐式时间积分器：

```xml
<physics>
  <ode>
    <implicit_spring_damper>true</implicit_spring_damper>
  </ode>
</physics>
```

- **效果**：它强制求解器将限位约束转换为隐式方程，保证关节运动到机械硬限位时的数值稳定，彻底消除了限位撞击带来的爆飞异常 。

### 5.3 虚拟物理拉簧机制（断电缓慢下坠，代码已被删除）

为了实现机械臂在不通电、断电状态下依然能保持适当折叠或者缓慢下坠，我们在关节中利用内置弹簧物理参数进行自平衡支撑，此段代码已被删除，效果与阻尼，摩擦力取代：

- **大臂俯仰关节 (link1_to_link2)**：

```xml
<dynamics>
  <damping>0.5</damping>
  <friction>0.5</friction>
  <spring_reference>-1.2</spring_reference>      <!-- 弹簧平衡位置：拉回折叠角度 -->
  <spring_stiffness>15.0</spring_stiffness>      <!-- 弹簧刚度：提供拉力对抗机械臂重力 -->
</dynamics>
```

- **小臂俯仰关节 (link2_to_link3)**：

```xml
<dynamics>
  <damping>0.5</damping>
  <friction>0.5</friction>
  <spring_reference>3.14159265</spring_reference>  <!-- 弹簧平衡位置：拉回 180° 折叠 -->
  <spring_stiffness>10.0</spring_stiffness>
</dynamics>
```

- **效果**：无需控制代码，仅靠 Gazebo 内置的弹性力学解算就能像物理拉簧一样将机械臂牢牢提拉在无人机下方，保证在断电情况下整机稳定垂直于地面，且 RTF 完美维持在 0.99 ~ 1.0。

## 六、Gazebo 调试技巧与状态观测

### 6.1 模型内置的关节手动控制器 (Joints Panel)

在 Gazebo Classic 11 界面中：

1. 将鼠标指针移动到渲染视窗的最右侧边缘。
2. 看到鼠标光标变成双向拉伸箭头时，按住鼠标左键向左拉拽，即可拉出隐藏的 Right Panel（右侧面板）。
3. 在 3D 视窗或左侧 Models 树中左键点击一下无人机模型 roarm_quad 将其选中joint。
4. 右侧面板将自动刷新并展现 Force、Position 和 Velocity 三个控制标签页。在 Position 中即可通过微调输入或直接滑动滑块，手动调节关节角度。
   （注意：如果模型处于 `<static>true</static>` 状态下，所有的关节物理解算会被冻结，此时 Joints Panel 将无法控制关节转动，滑块也会消失，必须在 `<static>false</static>` 时控制。）

### 6.2 实时质心观测 (CoM 漂移)

- 顶部菜单栏勾选 View → Center of Mass。
- **画面变化**：
  - 每个连杆上会出现表示各自质心的小球。
  - 无人机中心会出现一个带有红、绿、蓝三维轴线的大紫色球体，此球体即为整机的系统总质心 (Combined Center of Mass)。
  - **质心观测**：在静态悬空或平地状态下手动调节关节时，可以直观地观察到总质心球体随着机械臂的运动而在空中实时漂移和滑动。

### 6.3 物理引擎底层 Bug 诊断：world→Reset() 关节崩溃发散

- **现象描述**：在 Gazebo 中，如果您直接修改 SDF 并加载，模型在纯 Gazebo 下工作完美。但只要与控制系统或某些复杂的重置命令结合，点击"重置时间"或在启动第一帧，关节会瞬间严重偏转、变形、甚至将模型弹飞，超限位乱动。
- **机理解释**：这是 ODE 物理引擎的底层同步 Bug。当外界控制接口在第一帧连接时会调用 world→Reset()，或者在 lockstep 强制重置状态下，求解器重置时未清除上一帧累积的硬限位误差力矩 [1.1.1, 1.2.4]，导致关节在重置的一瞬间遭受巨大的奇异加速度抗衡，强行撕裂关节约束 [1.1.1, 1.2.4]。这与模型参数无关，是 Gazebo 自身的重置漏洞。
- **回避方案 (Delayed Spawning)**：
  不要在 Gazebo 的 .world 文件中原初始化时直接写入您的模型，也不要在未仿真前投递模型。
  - **步骤**：先启动一个纯净、空的 Gazebo 物理世界，等物理世界稳定并完成第一次重置和时钟同步后，再通过 ROS 2 的 spawn_entity 节点或 Gazebo 命令行，在启动 1~2 秒后将模型动态投递进去。
  - **效果**：此时模型避开了第一帧的重置冲击，能够 100% 保持完美的初始装配姿态，稳定运行。
